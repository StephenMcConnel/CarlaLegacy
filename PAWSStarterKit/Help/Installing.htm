1 Requirements

The PAWS Starter Kit uses the new XML/XSLT technologies in conjunction with the Internet Explorer browser. While this has greatly sped up the development time, it does mean you have to have certain software available on your machine.

To use the PAWS Starter Kit, you will need to have:

  • Internet Explorer 5.0 or 5.5.

    If you have IE 5.5, then you may also need the MSXML3 package.

    If you have IE 5.0, then you will also need to have:

    1. The Windows Installer (instmsi.exe) and
    2. The MSXML3 package (msxml3.exe)

  • The PAWS Starter Kit install package which consists of these files:
    • setup.exe
    • setup.lst
    • PAWSSt1.CAB
    • PAWSSt2.CAB
    • PAWSSt3.CAB

2 Installation

2.1 For Internet Explorer 5.5

Unless you already have the latest version of the MSXML3 package installed, then you will need to install it. You install it by invoking the msxml3.exe program and following the instructions on the screen. You may also need to run the Xmlinst.exe program to ensure that it will use the latest version (and not some earlier version) of the MSXML3 package.

2.2 For Internet Explorer 5.0

You must first install the windows installer, then the MSXML3 package before installing the PAWS Starter Kit. Follow these steps once you have the instmsi.exe and msxml3.exe programs:

  1. Install the Windows Installer by invoking the instmsi.exe program and following the instructions on the screen.
  2. Install the MSXML3 package by invoking the msxml3.exe program and following the instructions on the screen.

2.3 For the PAWS Starter Kit

Invoke the setup.exe program and follow the instructions on the screen.

3 Running the PAWS Starter Kit

The PAWS Starter Kit is a windows program like most other windows programs. You can find it by going to Start / Programs / PAWS Starter Kit / PAWS Starter Kit. You can also create a desktop short-cut to it. One way to do this is to navigate via the Windows Explorer to where you placed the program (the default is C:\Program Files\PAWSStarterKit). Go to the Container subdirectory. Right click on PAWSStarterContainer.exe and select Create Shortcut. Then drag the shortcut to your desktop.

3.1 First time running the PAWS Starter Kit

The very first time you run the PAWS Starter Kit, it will give a message saying that it is creating a new PAWS language project. There are three steps in this:

  1. You will be asked to enter the name of the language and a three letter abbreviation for it (this can be like what you do for CarlaStudio).
  2. You'll then be given a chance to select the font you want to use for your vernacular data.
  3. After that you will be asked to save the data in a file. We suggest you put the data in some directory other than the one where it will initially default to (which is where the PAWS Starter Kit stores its invariable data). It could even go in the same directory where your CarlaStudio data is, if you want.

After these initial steps, you will see the main screen, displaying the table of contents. In theory, it is self-explanatory. You'll have to tell us, though, whether the theory is right...

We do suggest that you initially check out the purpose statement. (It is the "Click here to see a purpose statement." near the top.)

3.2 Marking words in the Lexicon

Throughout the PAWS Starter Kit pages, you'll see instructions in red. When these say things like, "mark these words as Deg modifies_NP in the lexicon," it means to add a field to your appropriate AMPLE dictionary record(s) which contains the words "Deg modifies_NP".

The standard code that CarlaStudio produced by default is "\f" (for "features"). If you use this code, you would key the field like this:
\f Deg modifies_NP

You can certainly use whatever code you wish. Just make sure that you enter your code in all the dictionary SFM marker mappings in the language model portion of CarlaStudio (language model / dictionary tab).

3.3 Using PC-PATR in CarlaStudio

At some point we assume you will want to try and use the results of the PAWS Starter Kit with your CarlaStudio set up. Here is what we suggest:

  1. In CarlaStudio, go to the language model / Language / Analysis Sequence...
  2. Click on the last process in the list (which may well be SENT Disambig) and then click on the Insert After button. (By the way, you will probably want to disable the SENT Disambig process so you can better tell just how effective the syntactic approach is.)
  3. In the resulting Choose a Processor... dialog, click on PC-PATR DLL and then click the OK button.
  4. You should now see a PC-PATR Disambiguate processor in the list. Select this (it probably already is) and click on the Properties button.
  5. You will now see the PCPATR DLL dialog box. Click on the Browse button and navigate to where you have stored your PAWS Starter Kit data. Select the xxxPatr.grm file (where xxx is your language abbreviation).
  6. We suspect that you will want to keep the other options in the PCPATR DLL dialog as they are. You can check the online PCPATR reference manual if you want to know more about these options. (In CarlaStudio, go to Help / Help Topics / Other Documentation / Online documentation for the CARLA tools and then scroll down to where it has the PC-PATR Reference manual and click on the small grey box by it.)

The next time you process a file, it will also invoke PC-PATR and apply the rules created by the PAWS Starter Kit.

4 Special Notes for Version 0.2.4 Alpha

4.1 This is alpha software

This means that it is not complete (i.e. it does not have all the functionality we plan for it to have).

It also means we fully realize that we have not fully tested it (i.e. we won't be shocked if there are bugs). Please make a list of all problems, questions, and suggestions you have on it. Your feedback is greatly appreciated!

4.2 The Help File

In this version there is only one help file (other than this one). It is a WinWord version of Cheri's Introduction to GB that appeared in several volumes of Notes on Linguistics. There are several places in the PAWS Starter Kit where it says to "Click here to see more on X". Clicking on that link will have the program load in Cheri's file and jump to the appropriate section (assuming you have WinWord installed on your machine).

Note for IE 5.0 users: This seems to work fine on machines running IE 5.5. On machines with IE 5.0, however, it ends up crashing the PAWS Starter Kit. At this point, I do not know why it behaves this way. If you have IE5.0, you can always just use WinWord to edit the file (it is in the Help subdirectory). You can then use Edit / Go To / select bookmarks and choose the one that sounds like the one you're interested in (you can ignore all the xDDD example number bookmarks).

4.3 Auxiliary Tools

The PAWS Starter Kit makes mention of three auxiliary tools. Unfortunately, only one of them exists as of May 4, 2001.

4.3.1 The Log Browser Tool

Every section (except the first) ends with a Testing page. This page mentions a tool called the Log Browser. Morris Johnson of PNG is currently developing this tool. As of May 4, 2001, version 0.2a (alpha) is available. The Log Browser is designed to make the output of PC-PATR much easier to read. Click here to read an introduction on the Log Browser tool. Note that it requires IE 5.5 to use it.

4.3.2 The Category Mapper Tool

Some sections mention a Category Mapper tool. It does not yet exist. We are hoping that Morris will work on this after he completes the Log Browser. This tool is intended to simplify the process of making the categories you've used coincide with the ones the PAWS Starter Kit grammar file will assume.

Beginning with version 0.2.3, we've added a temporary solution. Create a file called XXXCatMap.txt and put it wherever your XXXPatr.grm file is (for both files, XXX means whatever language abbreviation you are using). This file will get appended to the XXXPatr.grm file whenever you save your PAWS language project.

In the XXXCatMap.txt file, type entries like this:
Let xxx be  <cat> = !Num
    <head type ordinal>  = !-
    <head type wh>   = !-
 where xxx is whatever your AMPLE category is (case sensitive) and the category after the exclamation point is whatever it needs to be according to the starterkit. The other lines are the default values per the Let statement in the XXXPatr.grm file. This tells the program to map whatever your category was to the category Num plus all the appropriate defaults.

This type of mapping will have to be done for every AMPLE category you have that either doesn't already match the syntactic category or isn't mapped by a feature.

4.3.3 The Feature Template Editor Tool

This is another hoped-for tool to make managing PC-PATR's templates easier.
