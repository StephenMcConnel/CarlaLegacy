PcPatrBrowser

Alpha Version 0.1
19-Sep-2005

PcPatrBrowser is a tool to aid in studying syntactic tree output from PcPatr.

This read me file is for Alpha version 0.1.

To make maixmium use of this tool, you should install the latest version of PCPATR.  In particular, you should
  1. Install Patr100.dll version 1.3.10.1 in your CarlaStudio program files directory.
  2. Do Start / Run, type
    regsvr32.exe "C:\Program Files\SIL\CarlaStudio\Patr100.dll"
    (or whereever you have it) and then click on OK.  This will register this new version of PCPATR.
This new version of PCPATR does several things:
  1. When you use the XML tree format option, it will:
    1. output the tree information in a form that PcPatrBrowser can read;
    2. output the location of the grammar file that was used in both the ANA file and the log file; and
    3. output information about the words and their analyses.  This latter is used in the interlinear view in PcPatrBrowser.
Note that if you use the XML tree format option, then you never need to manually tell PcPatrBrowser about the grammar file used. It is also the case that when you use the XML tree format option, all feature structures are always written to the output file.

Language Information file:
The only file that PcPatrBrowser creates is a language information file.  This file contains font information and an indication of whether or not the parse tree and interlinear should be displayed with a right-to-left orientation.  The first time you run PcPatrBrowser with data for a given language, you will probably want to create this language information file.  From then on, PcPatrBrowser ought to remember the language file you used last time.

Known problems:
  1. There is no user help for this alpha version.
  2. Fonts:
    1. Bold, italic, underline, etc. do not work.
    2. The interlinear view uses
      1.  the non-terminal font for
        1. category
        2. feature descriptors and
        3. format information
      2. the lex font for
        1. decomposition
        2. original word and
        3. underlying form information
      3. the gloss font for
        1. gloss information.
      4. There is no current way to control these fonts individually.
  3. The new version of PCPATR assumes a morpheme decomposition separation character of hyphen ('-').  If you are using something else, then the interlinear view will not come out correctly.  Sorry.
